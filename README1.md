**React.js এ অনেকগুলো গুরুত্বপূর্ণ ধারণা আছে, যেগুলো শিখে তুমি React ভালোভাবে ব্যবহার করতে পারবে। মূলত React শেখার জন্য ১০–১২টি প্রধান কনসেপ্ট সবচেয়ে জরুরি।**  

---

## 🔑 React.js এর প্রধান বিষয়গুলো (Bangla ব্যাখ্যা)

### ১. JSX  
- JavaScript + HTML এর মিশ্রণ।  
- এর মাধ্যমে তুমি HTML ট্যাগের মতো কোড লিখতে পারো, কিন্তু ভেতরে JavaScript ব্যবহার করতে পারো।  

### ২. Components  
- React অ্যাপের মূল বিল্ডিং ব্লক।  
- প্রতিটি কম্পোনেন্ট আলাদা ফাংশন বা ক্লাস হতে পারে।  
- ছোট কম্পোনেন্ট (যেমন বাটন) থেকে বড় কম্পোনেন্ট (যেমন পুরো পেজ) তৈরি করা যায়।  

### ৩. Props  
- কম্পোনেন্টে ডেটা পাঠানোর উপায়।  
- প্যারেন্ট কম্পোনেন্ট থেকে চাইল্ড কম্পোনেন্টে ডেটা পাঠাতে ব্যবহার হয়।  

### ৪. State  
- কম্পোনেন্টের ভেতরে ডেটা ম্যানেজ করার সিস্টেম।  
- ইউজার ইন্টারঅ্যাকশন বা ইভেন্ট অনুযায়ী UI আপডেট হয়।  

### ৫. Event Handling  
- যেমন `onClick`, `onChange` ইত্যাদি।  
- ইউজার কোনো কাজ করলে React সেই অনুযায়ী ফাংশন চালায়।  

### ৬. Conditional Rendering  
- শর্ত অনুযায়ী UI দেখানো বা লুকানো যায়।  
- উদাহরণ: যদি ইউজার লগইন করে থাকে তবে প্রোফাইল দেখাও, না হলে লগইন ফর্ম দেখাও।  

### ৭. Lists এবং Keys  
- ডেটার লিস্ট দেখানোর জন্য `.map()` ব্যবহার হয়।  
- প্রতিটি আইটেমের জন্য `key` দেওয়া জরুরি, যাতে React দ্রুত আপডেট করতে পারে।  

### ৮. Hooks (যেমন useState, useEffect)  
- Functional component এ state এবং lifecycle ফিচার ব্যবহার করার উপায়।  
- `useState` → state ম্যানেজ করে।  
- `useEffect` → সাইড ইফেক্ট (API কল, ডেটা ফেচ) হ্যান্ডেল করে।  

### ৯. Context API  
- Props drilling এড়ানোর জন্য ব্যবহার হয়।  
- একাধিক কম্পোনেন্টে ডেটা শেয়ার করা সহজ হয়।  

### ১০. Lifecycle Methods (Class Component এ)  
- কম্পোনেন্ট কখন তৈরি হচ্ছে, আপডেট হচ্ছে বা মুছে যাচ্ছে তা নিয়ন্ত্রণ করে।  

### ১১. Routing (React Router)  
- একাধিক পেজ বা ভিউ তৈরি করতে ব্যবহার হয়।  
- Single Page Application (SPA) এ নেভিগেশন সহজ করে।  

### ১২. Performance Optimization  
- Memoization (`React.memo`, `useMemo`, `useCallback`) ব্যবহার করে অপ্রয়োজনীয় রি-রেন্ডার কমানো যায়।
